# Android host

Native **host app** for Offline Games Hub on local Wi‑Fi.

A phone with this APK starts the server. Other players join with a **browser** (no guest APK).

The implementation lives in this folder: Compose UI, foreground service,
HTTP/HTTPS + WebSocket host, QR sharing, settings, and bundled web packs.

---

## Role

| Host does | Host does not |
|-----------|----------------|
| Start / stop local HTTP + WebSocket | Draw complex gameplay (that is HTML) |
| Show one simple QR invitation | Cloud matchmaking |
| Foreground service so the OS keeps the process | Force guests to install an app |
| UI language, keep-screen-on | Ship a full Node runtime |
| Serve lobby + game/program packs from `games/` | |

## Size goals

Product philosophy: **maximum gameplay per kilobyte**.

| Artifact | Target |
|----------|--------|
| Current debug APK (full catalog) | ~**18 MB** |
| Bundled web library | ~**11 MB** |
| One game pack | hard max **10 MB**, prefer **&lt; 2 MB** |
| Ultra badge | **&lt; 200 KB** |

Do not ship: Node, Chromium, Flutter engine, Unity, giant multilingual dictionaries in base.

## Build

```bash
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
cd android
./gradlew :app:assembleDebug
# APK → app/build/outputs/apk/debug/
```

`preBuild` runs `syncWebAssets`, which copies the complete `games/` tree and
`pc/www/`, excludes templates and bundled fonts, and places programs at
`assets/web/games/programs/<id>/`.

When building from the linked Android worktree, point it at the primary
checkout to package its current game library:

```bash
./gradlew :app:assembleDebug -PoghWebRoot=/path/to/OFFline_games_app
```

Canonical program URLs are `/games/programs/<id>/client/`. Legacy
`/programs/<id>/client/` requests redirect to the canonical path.

## Player flow

1. **Home** — one large “Play together” button and three short steps.
2. **Invite** — one QR code, share/copy actions, and “Play on this phone”.
3. **Help** — hotspot instructions only when they are needed.
4. **Settings** — language and screen behaviour first; port and HTTPS stay under Advanced.

The foreground service owns the real running state. Reopening the app or tapping
its notification therefore returns to the invitation instead of accidentally
showing a second start screen.
